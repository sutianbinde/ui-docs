    # Tree 树

    基本要求

    • 树的父级一定要有固定高度 或固定百分比 或flex布局(原理：内部通过clientHeight获取高度),如果没有获取到高度，将会默认渲染树高度为300px。
    • 如果树的父级开始是隐藏的（display:none），那么渲染的时候也会因为获取不到父级高度而渲染树高度为300px，
    • 修正方法1：在显示完成的时候，调用resize方法强制重新渲染
    • 修正方法2：通过 content-height 传入一个固定的高度数值（如 400) 来覆盖默认的 300px的渲染。
    • 修正方法3：不使用可视区渲染，使用render-all渲染全量的数据(数据量1000以内)
    • 如果将树放入 popover 会出现，隐藏显示后滚动距离重置问题，需要在每次重新显示的时候调用 调用resize 重置滚动位置
    • 关于闪动
    • 每个node 节点禁止换行，否则可能会出现滚动闪动
    • 避免出现横向滚动条，否则有可能出现滚动闪动， 建议通过 solt显示的节点，添加 v-tooltip

    # 基本用法

    通过data属性传递数据， 通过show-search属性设置是否显示默认检索栏, 通过expand-on-click-node属性设置是否点击行时展开收起，通过tree-update-ready事件可以监听树内部数据处理完成事件，通过node-click事件监听点击节点。

const findSourceNodeById = (sourceNodes, id, callback) => {
  sourceNodes.forEach((sourceNode, index, arr) => {
    if (sourceNode.id === id) {
      return callback(sourceNode, index, arr);
    }
    if (sourceNode.children) {
      return findSourceNodeById(sourceNode.children, id, callback);
    }
    return false;
  });
};

    computeMoveNeededParams(sourceNodes, draggingId, targetId, type) {
      let draggingSourceNode;
      let hasSameLevelNodesAsDraggingNode;
      let draggingNodeIndexAtSameLevelNodes;
      findSourceNodeById(sourceNodes, draggingId, (sourceNode, index, arr) => {
        hasSameLevelNodesAsDraggingNode = arr;
        draggingNodeIndexAtSameLevelNodes = index;
        draggingSourceNode = sourceNode;
      });
      let hasSameLevelNodesAsTargetNode = null;
      let targetNodeIndexAtSameLevelNodes;
      if (type === 'inner') {

          targetSourceNode = sourceNode;

        findSourceNodeById(sourceNodes, targetId, findSourceNodeCallback);
        return {
          targetSourceNode,
          originSourceNode: draggingSourceNode,
          originSourceNodeIndex: draggingNodeIndexAtSameLevelNodes,
          originSourceNodes: hasSameLevelNodesAsDraggingNode,
        };
      }
      const findSourceNodeCallback = (_, index, nodes) => {
        hasSameLevelNodesAsTargetNode = nodes;
        targetNodeIndexAtSameLevelNodes = index;
      };
      findSourceNodeById(sourceNodes, targetId, findSourceNodeCallback);

      return {
        targetSourceNodes: hasSameLevelNodesAsTargetNode,
        targetSourceNodeIndex: targetNodeIndexAtSameLevelNodes,
        originSourceNode: draggingSourceNode,
        originSourceNodeIndex: draggingNodeIndexAtSameLevelNodes,

      self.$message({
        message: '拖拽信息已打印到控制台',
        type: 'success'

      if (!fromItem || !toItem) return;
      const { id: draggingId } = fromItem;
      const { id: targetId } = toItem;
      // const sourceNodes = [...this.treeData];
      const {
        targetSourceNode,
        targetSourceNodeIndex,
        targetSourceNodes,
        originSourceNode,
        originSourceNodeIndex,
        originSourceNodes
      } = this.computeMoveNeededParams(this.treeData, draggingId, targetId, type);
      if (type === 'inner') {
        targetSourceNode.children = targetSourceNode.children || [];
        targetSourceNode.children.push(originSourceNode);
        originSourceNodes.splice(originSourceNodeIndex, 1);
      }
      if (type === 'before') {
        self.insertToBefore(
          targetSourceNodeIndex,
          targetSourceNodes,
          originSourceNode,
          originSourceNodeIndex,
          originSourceNodes
        );
      }
      if (type === 'after') {
        self.insertToAfter(
          targetSourceNodeIndex,
          targetSourceNodes,
          originSourceNode,
          originSourceNodeIndex,
          originSourceNodes
        );
      }
      self.$set(this.treeData, 0, this.treeData[0]);
    },
    nodeDrop(fromItem, toItem, type, event) {
      console.log('拖拽来源（外部拖入为 undefined）:', fromItem, '目标节点:', toItem, '类型:', type, 'drop 事件:', event);
      this.$message({
        message: '拖拽信息已打印到控制台',
        type: 'success'
      });
      if (!fromItem || !toItem) return;
      const newArr = [...this.treeData];
      let itemStartIndex = 0;
      let itemEndIndex = 0;
      let flag = false;
      newArr.forEach((item, index) => {
        if (fromItem.id === item.id) {
          flag = true;
          itemStartIndex = index;
        } else if (fromItem && fromItem.z >= item.z) {
          flag = false;
        }
        if (flag && fromItem.z < item.z) {
          itemEndIndex = index;
        }
      });

      const fromNumber = itemEndIndex ? itemEndIndex - itemStartIndex + 1 : 1;
      const fromItems = newArr.splice(itemStartIndex, fromNumber);
      let { parentId } = toItem;
      let defaultZDistance = toItem.z - fromItems[0].z;
      let startIndex = newArr.indexOf(toItem);


      if (type === 'inner') {
        parentId = toItem.id;
        defaultZDistance += 1;
        startIndex += 1;
      } else if (type === 'before') {
        //
      } else if (type === 'after') {
        startIndex += 1;
      }
      fromItems[0].parentId = parentId;
      fromItems.forEach((item) => {
        item.z = defaultZDistance + item.z;
      });
      newArr.splice(startIndex, 0, ...fromItems);
      this.treeData = newArr;
    }
  }

    node-key 作为 id 的替代项，将会以传入的 key 名称作为 node id 的映射 String — "id"
    default-expanded-keys 默认展开的节点的 key 的数组, 传入非空数组，将会只展开传入项及父级,出于性能考虑，只接收长度不大于30的数组 Array — []
    default-expand-all 是否展开全部(会被default-expanded-keys 覆盖) boolean — true
    item-class-fn(node) 根据返回值为节点添加独立的 class Function — -
    check-on-click-node 是否在点击节点的时候选中节点，默认值为 false，即只有在点击复选框时才会选中节点。 boolean — false
    filter-node-method(value, node) 对树节点进行筛选时执行的方法，返回 true 表示这个节点可以显示，返回 false 则表示这个节点会被隐藏 Function — -
    search-placeholder 检索框的 placeholder String — "请输入关键字进行检索"
    record-dynamic-select 显示选中的统计数据 boolean — false
    show-dept-icon 显示机构或区域图标（需提前在传入数据中添加 flag: 'dept'/'place' 表示是机构/区域） boolean — false
    show-camera-icon 显示机构图标（需提前在传入数据中添加 flag: 'camera' 表示是相机） boolean — false
    show-search 是否显示检索栏 boolean — true
    clearable 检索栏 输入框可清空 boolean — false
    item-height 每个节点的高度 number — 34
    search-height 检索栏高度 number — 44
    lazy 是否懒加载子节点，需与 load 事件结合使用 boolean — false
    search-leaf-model 检索叶子节点模式 检索的时候，只检索非noStatistics的叶子节点，并以列表形式呈现选择结果，注意：此模式下，点击全选时候会触发 check-change事件，但是参数为空 boolean — false

    # 方法

    方法名 说明 参数
    filter 对树的 label 进行查找显示 (searchText) 用于查找的文本，支持大小写混查
    getCurrentKey 获取当前被选中节点的 id，若没有节点被选中则返回 null -
    setCurrentKey 通过 key 设置某个节点的当前选中状态 (key) 待被选节点的 id，若为 null 则取消当前高亮的节点
    getCurrentNode 获取当前被选中节点的 data，若没有节点被选中则返回 null -
    getCheckedNodes 若节点可被选择（即 show-checkbox 为 true），则返回目前被选中的节点所组成的数组 (filter方法, leafOnly) filter 会传入选中节点对象， 会根据返回值判断是否返回当前节点, leafOnly只返回子节点的
    getCheckedKeys 若节点可被选择（即 show-checkbox 为 true），则返回目前被选中的节点的 id 所组成的数组 (filter方法, leafOnly) filter 会传入选中节点对象， 会根据返回值判断是否返回当前id, leafOnly只返回子节点的 id
    setCheckedNodes 通过 node 设置目前勾选的节点 ( nodes)，要勾选的节点数组，如果想取消全部选中，可传[]， 全选传"ALL"
    getHalfCheckedNodes 若节点可被选择（即 show-checkbox 为 true），则返回目前被半选中的节点所组成的数组 (filter方法) filter 会传入选中节点对象， 会根据返回值判断是否返回当前节点
    getHalfCheckedKeys 若节点可被选择（即 show-checkbox 为 true），则返回目前被半选中的节点的 id 所组成的数组 (filter方法) filter 会传入选中节点对象， 会根据返回值判断是否返回当前id
    setCheckedKeys 通过 ids 设置目前勾选的节点 ( ids)，要勾选的id数组，如果想取消全部选中，可传[]， 全选传"ALL"
    resize 重新计算树的高度和显示的首末位置 (height)期望渲染的高度，如果不传，就以树父级100%来获取高度，如果100%高度获取到的值为0，再以content-height（默认300）高度渲染
    setChecked 根据 key 设置某个节点的 选中状态 (key, value) （id, true或false）
    setNodeAttributeByKey 根据 key 设置某个节点的 某个属性 的值 (key, name, value) （id, 属性名，属性值）
    getNode 通过 id 获取节点 -
    setChildrenChecked 根据 parentKey 设置所有子节点的 选中状态（对于部门选择的变态关联十分有效） (parentKey, value) （id, true或false）
    setData 手动设置树数据，不形成 vue 的数据监听，对大数据适用 (data) （和参数中的 data 一致）

    # Events

    事件名 说明 参数
    node-click 节点被点击时的回调 共两个参数，依次为：当前节点对象、节点的源数据
    check-change 节点选中状态发生变化时的回调 共两个参数，依次为：当前节点对象、节点本身是否被选中
    node-drop 拖拽放手时触发的事件(不会真的移动节点，只是推出事件，具体数据层次操作需外部手动完成), 如果是树以外的地方接收，通过e.dataTransfer.getData('draggingData')可获取到拖拽的节点的源数据(已转化为 String 类型)) 共四个参数，依次为：被拖拽节点对应的源数据、结束拖拽时最后进入的节点的源数据、被拖拽节点的放置位置（before、after、inner）、event
    tree-update-ready 树节点渲染完成触发 一个参数：全部的节点数据
    node-drag-start 拖拽开始 共两个参数，依次为：被拖拽节点、event
    node-drag-end 拖拽结束 共两个参数，依次为：被拖拽节点、event
    load 加载子树数据的方法，仅当 lazy 属性为true 时生效 共两个参数，依次为：node节点对象、resolve()回调，回调函数的参数为新子节点数组
    after-filter 当树发生筛选查找完成触发 nodes 筛选之后显示的节点
    filtered-nodes-all-check-change search-leaf-model为true时，点击全选时触发 两个参数，依次为：nodes 筛选之后的数据中选中状态发生改变的节点， checked 是否选中
